새 세션으로 인계하기 전 마무리 루틴이다. 아래를 **순서대로** 수행하고, 각 단계 끝에 한 줄로 결과만 보고해라. 군더더기 설명 금지.

## 1. 현재 상황 최신화 + 기록
- `PROGRESS-BOARD.md`를 읽고, 이번 세션에서 바뀐 내용으로 갱신한다.
  - 끝난 **잎(Task)**은 보드에서 지우고 다음 것으로 교체.
  - 숲(로드맵)·나무(현재 plan) 골격은 그 단위가 끝난 게 아니면 그대로 둔다. 하지만, 골격이 달라질 경우 교체한다.
- 보드 맨 위에 "다음 세션 시작점" 한 줄을 남겨라 — 새 창이 곧바로 이어받을 수 있게.

## 2. 메모리 저장 (다음 세션이 알아야 할 것만)
- 코드/깃 히스토리로 알 수 있는 건 적지 마라. **비자명한 결정·맥락·막힌 지점만.**
- 기록 위치: `PROGRESS-BOARD.md` 상단 + 필요시 `CLAUDE.md`의 관련 섹션.
- (Cowork 자동 메모리는 이 명령과 별개다. 거기 남길 건 Cowork 세션에서 "기억해줘"로 따로 처리.)

## 3. 커밋 및 푸시
- `git status`로 변경 확인 → `git add -A`
- 커밋 메시지: 이번 세션 변경의 **의도** 중심 한 줄 (인자로 요약 주어지면 그걸 반영: $ARGUMENTS)
- `git push`
- API 키·시크릿이 diff에 평문으로 들어갔는지 푸시 전 반드시 확인. 있으면 중단하고 보고.

## 완료 보고
세 단계 결과를 각 한 줄씩. "다음 세션 시작점" 문구를 마지막에 다시 출력.
