Design Decisions

왜 이렇게 짰나

왜 kind는 "layout/component" 두 가지만?

AI가 카테고리를 만들지 못하게

"그래프 vs 차트 vs 도표" 같은 자유 분류 난립 차단. 사람이 정한 고정 enum에 LLM이 배정만 한다. 나머지 분류는 벡터 의미 검색이 처리한다.

왜 metadata를 jsonb로?

스키마 진화에 자유롭게

colors·fonts·slots 같은 서브필드를 DB 컬럼 추가 없이 확장 가능. 추천에 필요한 필드(embed_text·embedding)만 명시적 컬럼으로 분리, 나머지는 jsonb에 넣는다.

왜 embed_text와 embedding을 분리?

재임베딩 비용을 0으로

임베딩 모델이 바뀌거나 embed_text 포맷을 개선하면 embed_text 컬럼에서 바로 재생성 가능. 전체 이해 파이프라인을 다시 돌릴 필요가 없다.

왜 AssetColor에 role·locked를?

팔레트 자동화 + 보호 동시에

role로 "main/text/background"를 구분해 PaletteRoleMapper가 자동 팔레트를 만들 수 있다. locked=true인 색은 자동 팔레트가 건드리지 않는다.

왜 스타일 팔레트는 LLM 없이?

속도 + 비용 + 예측 가능성

WCAG 대비비·HSL 수학은 결정적(deterministic)이다. LLM을 쓰면 같은 에셋에서 매번 다른 팔레트가 나올 수 있고, 토큰 비용도 든다. 색상 수학이 더 정확하다.

왜 IAccessPolicy를 인터페이스로?

유료화 전환 비용을 0으로

지금은 admin.json 파일 존재 여부로 판단. 유료화 시 Supabase Auth 로그인 role 확인으로 교체만 하면 된다. 인제스트 코드·UI 코드 한 줄도 안 바뀐다.
