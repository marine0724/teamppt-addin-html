# TEAMPPT 발표 자산 (누적 기록)

> 대표·투자자 발표용. **기술 자랑이 아니라**, 비전문가가 *가치·낮은 위험·큰 성장 가능성*을 느끼고 감동받게 하는 게 목적.
> 빌드하면서 **완성된 기능**과 **버그 방지용으로 설계된 기능**을 *비전문가 언어로* 여기에 누적한다. (규칙: memory `feedback-presentation`)
> 발표자료가 필요하면 이 문서를 토대로 **도식화**해서 제시한다.

---

## 0. 한 문장 — 우리가 만드는 것

**바이브 디자이닝(Vibe Designing).** 커서가 코딩을 누구나 하게 만들었듯, TEAMPPT는 PPT 디자인을 그렇게 만든다 — *전문가의 손이 없어도 같은 결과물*.

## 1. 프로젝트 정체성 (왜 감동인가)

- **실제 절망과 정확히 맞물림:** "내용은 있는데 디자인이 안 된다." 콘텐츠는 이미 있으니 AI가 지어낼 게 없다(할루시네이션 0).
- **3단계로 점점 더 위임:** A 곁에서 도와주기(커서식) → B 통째로 맡기기(클로드코드식) → C 백지부터 기획까지. 지금은 토대인 **A**.
- **A만 써도 감동**이 목표. 토대가 단단하면 B·C는 그 위에 자연스럽게 얹힌다.

## 2. 완성된 기능 (시스템이 어떻게 도는가) — *빌드하며 누적*

> 도식화 대상. 각 항목은 "사용자가 뭘 하면 → 시스템이 뭘 해서 → 무엇이 보인다" 형태로.

- **AI가 에셋을 "본다":** 디자인 에셋(PNG)을 AI가 직접 보고, "이건 표지 레이아웃이다", "여기에 제목·부제·이미지 자리가 있다", "연도를 강조할 때 쓰면 좋다" 같은 구조화된 이해를 자동 생성한다. 사람이 하나하나 설명을 입력할 필요가 없다.
- **검색에 최적화된 의미 기록:** AI가 이해한 내용을 "투자 유치 IR 표지", "회사 소개 첫 장" 같은 실사용 의도 문장으로도 기록해, 나중에 사용자가 자연어로 검색하면 정확히 맞는 에셋이 나온다.
- **에셋이 클라우드 DB에 살아 있다:** AI가 이해한 에셋 데이터를 클라우드 벡터 데이터베이스(Supabase)에 자동 적재한다. 에셋 이미지와 원본 파일도 함께 올라간다. 나중에 사용자가 "장점 나열할 에셋 추천해줘"라고 치면 의미 검색으로 가장 잘 맞는 에셋이 나온다. 수작업 json이 아니라 **LLM 인제스트→벡터 검색** 파이프라인이 돈다.
- **관리자만 에셋을 올릴 수 있다:** 인제스트(에셋 업로드)는 관리자 권한이 있는 PC에서만 동작한다. 일반 사용자는 읽기만 가능. → R&D가 에셋을 계속 붓되, 사용자가 DB를 오염시킬 수 없다.
- *(A-2 공유 엔진 완료 시 채움)*
- *(A-3 추천·진단 UX 완료 시 채움)*

## 3. 위험을 미리 막으려고 *설계된* 기능 (왜 안전한가) — *빌드하며 누적*

> "이런 사고가 날 수 있는데, 우리는 이렇게 설계해서 원천 차단했다."

- **비파괴 원칙:** 사용자의 원본 슬라이드는 절대 건드리지 않는다(복제본에만 적용). → 망칠 위험 0.
- **키 유출 방어:** 관리자 권한은 로컬 키 존재 여부로만, 유출 시 키 회전으로 즉시 무력화.
- **분류 체계 고정:** 에셋 종류(layout/component)는 사람이 정한 2가지로만 고정하고 AI가 임의로 카테고리를 만들지 못하게 막았다. → "그래프 vs 차트 vs 도표" 같은 혼란 원천 차단.
- **유료화 교체 지점 내장:** 관리자 판단("이 PC가 관리자인가?")을 하나의 교체 가능한 모듈(IAccessPolicy)로 격리했다. 지금은 로컬 파일로 판단하지만, 유료화 때 로그인 기반으로 바꿔도 나머지 코드는 한 줄도 안 바뀐다. → 유료화 전환 비용 최소화.
- **DB 읽기/쓰기 완전 분리:** 일반 사용자는 읽기 전용(anon 키 + RLS 정책), 쓰기는 서비스 전용 키로만 가능. → 키가 유출되어도 DB를 망칠 수 없다.
- *(빌드하며 추가)*

## 4. 앞으로 진행될 기능 + 그에 따른 사용자 경험

- **화면 공유 진단 (A-2, 설계 완료·구현 예정):** 작업 중인 슬라이드를 두고 「화면 공유 진단」 버튼을 누르면, AI가 *지금 이 화면을 같이 보고 있는 것처럼* "공유 중" 표시를 띄우며 슬라이드를 직접 보고 개선점을 짚어준다. 그리고 사용자가 이어서 궁금해할 질문 3가지를 미리 만들어 둔다 — 클릭하면 같은 화면 맥락 그대로 대화가 이어진다. 화상회의에서 화면 공유하며 옆 사람에게 조언받는 경험을, AI로.
  - *왜 영리한가:* 슬라이드는 넘기긴 쉬워도 "읽기"는 비용이 든다. 그래서 매 순간 훔쳐보는 게 아니라, **사용자가 도움이 필요한 바로 그 순간 한 번** 보게 설계했다 — 사용자는 실시간 공유처럼 느끼지만 실제론 꼭 필요할 때만 읽어 비용을 최소화한다.
- **A 완성 후 → B(리디자인):** 초안을 열어두면 AI가 훑어보고 "리디자인 해드릴까요?" → OK 하면 컨셉만 고르면 통째로 바뀐다.
- **B→A 되돌이:** 리디자인 후 마음에 안 드는 부분은 에셋을 하나씩 갈아끼움.
- **C(기획):** 백지에서 구성 제안부터.

## 5. 성장 가능성 (왜 더 커지는가)

- **데이터가 해자:** 에셋이 LLM 인제스트→벡터 DB로 쌓일수록 추천이 정확해진다. R&D가 계속 붓는 구조.
- **같은 엔진 재사용:** A의 '읽기 엔진'이 B·C에 그대로 쓰임 → 한 번 만든 토대가 전 루트에 복리로 작동.
- **노멀 사용자 확장:** 지금은 약간의 안목이 필요하지만, 점점 누구나 같은 결과에 닿게.
