## 참고 (사람용 메모)

- **목적:** trace를 "독백 예쁘게"가 아니라 **"지금 뭐가 병목인지 가려주는 진단 계기판"**으로. 기능 문제가 사라질 때까지.
- **순서 주의:** Task 4에서 `DesignCritiqueService.CritiqueAsync` 시그니처가 바뀌므로 같은 커밋에서 `RecommendationService` 호출부를 고쳐야 컴파일됨. Task 5에서 `RunAsync` 시그니처가 바뀌므로 같은 커밋에서 `AssetPanel` 호출부를 고쳐야 함.
- **thinkingBudget:** 이해/구성 768, 검수 2048 (보수적 시작 — PPT 검증하며 조정). 나중에 Claude 엔진 교체해도 인터페이스 동일.
- **병목 4분류:** 기능 / 데이터추출 / 에셋부족(양) / 에셋품질(질). 검수자가 이해 요약 + 검색 유사도를 텍스트로 받아 종합 판정.
