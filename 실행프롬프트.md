빌드(반드시 관리자 권한, CLAUDE.md):

핵심 불변식 (어기지 말 것):
- LLM은 텍스트 내용을 생성/수정하지 않는다. 표시·이주 텍스트는 항상 COM 원문.

- 적용은 비파괴: Slide.Duplicate() 복제본에만. 원본은 어떤 실패에도 불변.
- 좌표 변환(CoordinateConverter)에 폴백 로직 추가 금지 — 실패는 드러낸다.
- 초안 읽기는 런타임 저가 1회 호출(gemini-2.5-flash). 비싼 이해는 인제스트(에셋) 쪽에 이미 끝나 있음.
- API 키를 문서·커밋에 평문으로 넣지 말 것.

이번 범위(YAGNI): 본문 레이아웃 슬라이드 1장 풀변환(추천 Top2→선택→재료 이주)까지만.
제외: 덱 전체 일괄, 이미지 미화, 컨셉/팔레트 재테마, 표/차트 슬롯 정교화.

코딩 직전 실제 시그니처 확인이 필요한 재사용 지점:

## 참고 (사람용 요약)

- **목표:** 초안 슬라이드를 LLM이 읽어 적합 에셋 Top 2 추천 → 선택 → 초안 재료를 에셋 자리에 실제로 채워 비파괴 변환.
- **생명:** 초안 데이터 추출 — 재료 종류·양·디자인을 매칭에 적합하게 담는 `DraftUnderstanding`.
- **신규 파일 13개 / 수정 2개**(GeminiAiService, AssetPanel). 플랜 Task 1~12.
- **단위테스트 가능(xUnit):** 모델/텍스트메트릭/이해스키마/이해파서/슬롯매핑파서. 나머지(COM·LLM·UI)는 수동 검증.
- **데모 임팩트 지점:** 시안 2개가 둘 다 "내 내용으로 채워진" 모습 + 원본(N)/선택(N+1) 비포애프터.
